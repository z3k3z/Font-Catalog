# Fontopia Build and Release Checklist

## Pre-Release Preparation

[ ] Verify application functionality from source.
[ ] Verify executable packaging still functions correctly.
[ ] Review outstanding TODO items and known issues.
[ ] Ensure all desired changes are committed to git.
[ ] Verify working tree is clean:

```
git status
```

## Generate Release Notes Draft

[ ] Run:

```
.\scripts\get-release-changes.ps1 > packaging\release-notes-draft.txt
```

[ ] Review generated change list.
[ ] Curate and summarize release notes.
[ ] Update:

```
packaging\README.txt
```

[ ] Verify README version placeholder exists:

```
{{VERSION}}
```

## Build Release Artifact

[ ] Run:

```
.\scripts\build-desktop.ps1
```

[ ] Enter requested version tag.
[ ] Verify successful PyInstaller build.
[ ] Verify generated ZIP exists:

```
dist\Fontopia-vX.Y.Z.zip
```

## Validate Release Artifact

[ ] Extract ZIP to a clean temporary folder.
[ ] Launch:

```
Fontopia.exe
```

[ ] Verify browser opens automatically.
[ ] Verify fonts enumerate successfully.
[ ] Verify search functionality.
[ ] Verify detail panel functionality.
[ ] Verify liked-font/cart functionality.
[ ] Verify About/build-info endpoint responds correctly.
[ ] Verify version/build metadata correctness.
[ ] Verify README.txt exists in deployment folder.
[ ] Verify no missing static assets or console errors.

## Create Git Tag

[ ] Run:

```
.\scripts\add-version-tag.ps1
```

[ ] Verify tag creation succeeded.
[ ] Push commits:

```
git push
```

[ ] Push tag:

```
git push origin <tag>
```

## Post-Release

[ ] Archive deployment ZIP if desired.
[ ] Distribute ZIP to testers/users.
[ ] Capture user feedback/issues against reported version number.
